Lesson 1: The Basics of Web and HTML

How the Web Works

The internet is the largest computer network that uses http to communicate between the server and the browser on your computer. The server is a computer that hosts files that makes up the web. The browser reads, interprets, and displays the information on your computer screen. This video explains this process.

HTML

HTML stands for hypertext marklup language. HTML documents form the majority of web content. HTML documents contains text content which describes what you see and markup which describes what it looks like. This video details this process.

Tags and Elements

HTML documents are made of HTML elements. Elements are comprised of tags which perform various functions. An example of a tag would be to italicize a font, the italic tag would require an open tag and a close tag with content inbetween. This open tag , context, and close tag would be called an element in the HTML document. This video demonstrates this.

Why Computers are Stupid

Computers are stupid because they interpret instructions literally. A small mistake by the programmer can cause a huge problem in a program.

Inline VS Block Elements

The two types of elements in HTML are either inline or block. Block elements form an invisible box around the content inside of them.

Avoiding Repetition

When writing HTML it is important to avoid repetition because repetition can lead to more errors in your code, including syntatic errors so it is important to be as consise as possible. By goruping your classes together you can easily change the styling of a class using CSS (described below) as a whole rather than indiviual div styling to avoid any copy and paste mistakes. To develop and maintain your code most efficiently it is important to create larger classes so you minimize any errors that could occur using repetition. Repetition = bad.

Lesson 2: Creating a Structured Document with HTML

Tree-Like Structure

HTML elements have other elements inside them. One element can have elements containing other elements. They are drawn out and displayed in a series of connected and nested boxes that portray a tree-like structure.

Indented HTML & Boxes

When you read the text for HTML you will see indentations at different points. The further indented an element is the more nested its box is.

Text Editors

Text editors are used to write HTML and they can help with shortcuts. An example can be seen using Sublime text because the text editor automatically generates a closing HTML tag.

Lesson 3: Using CSS to Add Design Styles

CSS

CSS stands for Cascading Style Sheets . CSS is used to add color, font, background, border, etc to your HTML design.

Key Terms

There are a few key terms to remember when writing HTML and CSS. Style can refer to a HTML element, HTML attribute, the general look of a page, or a verb. Rule is the line of CSS code describing the value a certain attribute takes. Property is the property you want to change. Value is the value you assign to the attribute. The selector is the name you use to target the element assigned to class in HTML. Class is a group of elements styled together (no punctuation). Finally, ID identifies an element in HTML.

Adding CSS to HTML

There are a few options on how you can add CSS to your HTML, but the recommended option is to write a separate file using CSS and link it to your HTML file. Adding a link element inside the head of HTML using the link tag will link the separate CSS file to the HTML. The link tag is a void tag so you will not have a closing tag. Example: link rel = "stylesheet" href "main.css" .

Divs

A div is a block element used to organize and separate HTML. You should use div as a opening and closing tag before and after block elements to help you visualize your blocks in HTML. Since div is a block element by default it takes up the entire width of a page, so to override this you will add code to your CSS file that will allow you to arrange the boxes (divs) next to each other. Example: display: -flex; in CSS will allow this.

Boxes and CSS

All boxes must be sized so that they diplay properly in HTML. The size of an element = left border + left padding + content + right padding + right border. Using this example: box-sizing: border-box; in CSS will make the box the same size, regardless of re-sizing the window. You can also set a max width for box sizes or set the box to a percentage of the window to allow conformity regardless of screen re-sizing.

Browser Specific Modifications in CSS

In order for your CSS elements to look the same in different browsers you should add these modifications to the code for box-sizing and flex. -ms, -webkit, -moz. An example: display: -webkit-flex; display: flex;.
